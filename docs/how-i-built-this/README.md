# How I Built This Notes

This shelf keeps implementation notes that are useful for Mark and future agents but
should not appear as customer-facing marketing copy.

Customer pages should explain what Bumpgrade does, why it is useful, and what a
publisher can do next. They should not lead with issue numbers, PR numbers,
contract IDs, source-data routes, framework names, database names, or build
status phrasing.

Use this shelf for:

- route contracts and source-data notes
- implementation caveats and operational blockers
- issue/PR/work-log references
- validation notes and screenshot policy details
- details about why a page was structured a certain way

Public docs for agents can still expose machine-readable contracts. The rule is
only that normal buyer pages should read like a product, not a build log.
