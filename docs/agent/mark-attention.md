# Mark Attention

Use this for important items Mark should not miss while agents keep working.

## Prefixes

Use `MARK ACTION REQUIRED` only when Mark must do something before progress can
continue.

Include:

- exact action needed;
- why it blocks progress;
- where Mark should report completion;
- issue/PR/session link;
- deadline or urgency if real.

Use `MARK ATTENTION REQUIRED` when work can continue but Mark should notice a
decision, risk, cost, tradeoff, or caveat.

## Do Not Block For Non-Blocking Attention

If the item is important but not blocking:

1. Record it in `/admin/roadmap`, `/admin/work-log`, an issue comment, or the
   agreed attention surface.
2. Email `m@rkmoriarty.com` if the project has no better attention channel yet.
3. Keep moving on independent useful work.
4. Check for replies before starting unrelated large work.

## Good Attention Item Shape

```text
MARK ATTENTION REQUIRED

Topic: [short title]
Why it matters: [one or two sentences]
Current decision: [what the agent did or plans to do]
Risk/tradeoff: [what Mark may want to revisit]
Where tracked: [issue/PR/admin URL]
No action needed to keep current work moving.
```
